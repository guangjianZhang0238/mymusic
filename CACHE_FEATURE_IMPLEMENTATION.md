# 音乐播放器缓存功能实现说明

## 功能概述
实现了APP端的歌曲播放缓存功能，播放过的歌曲会自动缓存在本地，下次播放时优先使用缓存内容，节省流量并提升播放速度。

## 技术实现

### 1. 缓存管理器 (CacheManager.kt)
- **核心功能**：提供全局唯一的缓存实例管理
- **新增功能**：
  - `generateCacheKey(songId: Long)`：根据歌曲ID生成唯一缓存键
  - `isSongCached(context: Context, songId: Long)`：检查歌曲是否已缓存
  - `getCachedSongInfo(context: Context, songId: Long)`：获取缓存歌曲详细信息
  - `CachedSongInfo`：缓存信息数据类，包含歌曲ID、缓存键、缓存大小等信息

### 2. 播放控制器 (PlayerController.kt)
- **新增方法**：
  - `playSongWithCache(song: SongDto, baseStaticUrl: String)`：智能缓存播放方法
  - `getSongCacheStatus(songId: Long)`：获取歌曲缓存状态
  - `isSongCached(songId: Long)`：检查歌曲是否已缓存

### 3. 视图模型 (MusicViewModel.kt)
- **修改方法**：
  - `playSong(song: SongDto)`：改为调用缓存播放方法
- **新增方法**：
  - `getCacheSize()`：获取格式化的缓存大小显示
  - `getCachedSongsCount()`：获取已缓存歌曲数量
  - `clearCache()`：清理缓存功能

### 4. 设置界面 (SettingsScreen.kt)
- **增强功能**：
  - 显示详细的缓存路径信息
  - 展示已缓存大小和歌曲数量
  - 添加缓存使用说明和提示
  - 保留原有的缓存清理功能

## 缓存策略

### 播放流程
1. 用户点击播放歌曲
2. 系统检查该歌曲是否已在本地缓存
3. 如果已缓存：直接从缓存播放（节省流量）
4. 如果未缓存：从网络加载并自动缓存（首次播放）

### 缓存管理
- 使用ExoPlayer内置的缓存机制
- 缓存目录：`/Android/data/com.music.app/cache/music_cache/`
- 最大缓存容量：500MB
- 自动LRU清理：存储空间不足时自动清理较旧内容

## 用户体验优化

### 设置界面改进
- **缓存路径可见**：用户可以看到具体的缓存存储位置
- **缓存统计**：显示已缓存大小和歌曲数量
- **使用说明**：提供清晰的缓存功能说明
- **清理功能**：保留一键清理缓存的便捷操作

### 性能优势
- **节省流量**：重复播放无需重新下载
- **提升速度**：本地缓存读取比网络加载更快
- **离线播放**：在网络不佳时仍可播放已缓存歌曲

## 注意事项

- 缓存文件存储在应用私有目录，卸载应用会自动清理
- 系统存储空间不足时会自动清理缓存
- 用户可以通过设置界面手动清理缓存
- 缓存机制对用户透明，无需额外操作
